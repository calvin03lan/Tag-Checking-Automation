# 格式
这个表格是测试信息的升级版，相比于原本的输入格式，它一行就可以储存包括DoublueClick、Gtag、Meta等不同种类Tag的信息。
所有的表头都在第3行，第4行及以后就是测试信息。以下是对于每一个对测试有用的列的描述：

# 每列的意义：
B-G：DoubleClick、Gtag、Meta、The Trade Desk、Taboola、Applier，每一列的值只能输入“Required” 和 “Not Required”，代表当前任务需不需要该种标签。
H：Page or event？ 该列的值只能输入“page” 和 “event”，代表是监测页面加载还是点击事件。
K：Language 可选“/en-hk、/zh-hk、/zh-cn”三个值。
L：URL path name，是完整URL的一部分。
P：Button ID，是button组件的唯一识别码，只有H列选择“event”才有用。

Q：type(Group Tag String)，是DoubleClick的识别副Keyword；
L：cat(Group Tag String)，是DoubleClick的识别主Keyword；
QL两列只有B是“Required” 的时候才有用。

S：Gtag Event snippet (Conversion ID / Conversion label)，其中的Conversion label是Gtag的唯一识别Keyword；
S列只有C是“Required” 的时候才有用。

T：Master Pixel ID，是Meta Pixel的识别副Keyword；
U：EV value，是Meta Pixel的识别主Keyword；
TU两列只有D是“Required” 的时候才有用。

V：Account ID，是The Trade Desk的识别副Keyword；
W：CT value，是The Trade Desk的识别主Keyword；
VW两列只有E是“Required” 的时候才有用。

X：Account ID，是Taboola的识别副Keyword；
Y：EN value，是Taboola的识别主Keyword；
XY两列只有F是“Required” 的时候才有用。

Z：type，是Applier的识别副Keyword；
AA：action_id，是Applier的识别副Keyword；
AB：track_id，是Applier的识别主Keyword；
这三列只有G是“Required” 的时候才有用。

# 新的读入逻辑
人工操作从点击Browse按钮开始，读入Excel表格。
在表格中，K、L、P 三列（语言、URL、Button ID）共同构成该表的三元复合主键，只有任意两行的这三者同时重复才算重复行，需要报错
读入逻辑：
1. 读入L列，解析出其中的不重复URL，并将其添加到URL管理部分，给这些URL按照原本在表格里的顺序自动编号，并根据所在行的K列赋予它们语言属性。
2. 对于每一行，检查B-G列是否为Required，如果有，则按照上面的对应规则读入主Keyword到Keyword管理中。
